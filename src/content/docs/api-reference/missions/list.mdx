## GET /missions

Retourne la liste paginée des missions de l'organisation active. Supporte le filtrage par statut, opérateur, client et date.

### Query params

| Paramètre | Type | Description |
|-----------|------|-------------|
| `status` | string | Filtre par statut. Valeurs : `PENDING`, `ASSIGNED`, `IN_PROGRESS`, `ON_SITE`, `COMPLETED`, `CANCELLED`. Peut être répété pour filtrer sur plusieurs statuts : `?status=PENDING&status=ASSIGNED`. |
| `operator_id` | string | Filtre les missions assignées à un opérateur spécifique. |
| `client_id` | string | Filtre les missions d'un client spécifique. |
| `from` | string | Date de début au format ISO 8601. Filtre sur `created_at`. |
| `to` | string | Date de fin au format ISO 8601. Filtre sur `created_at`. |
| `page` | integer | Numéro de page (défaut : `1`). |
| `per_page` | integer | Résultats par page (défaut : `20`, max : `100`). |

### Exemple de requête

```bash
# Missions en cours aujourd'hui
curl "https://api.dispatchkit.io/api/v1/missions?status=IN_PROGRESS&status=ON_SITE&from=2024-01-15T00:00:00Z" \
  -H "Authorization: Bearer {token}"
```
