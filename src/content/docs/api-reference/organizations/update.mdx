## PUT /organizations/`{id}`

Met à jour les informations d'une organisation. Seuls les champs fournis sont modifiés.

<Aside type="caution">
  Seuls les administrateurs de l'organisation peuvent effectuer cette action.
</Aside>

### Path params

| Paramètre | Type | Requis | Description |
|-----------|------|--------|-------------|
| `id` | string (UUID) | Oui | Identifiant de l'organisation. |

### Body

| Paramètre | Type | Requis | Description |
|-----------|------|--------|-------------|
| `name` | string | Non | Nouveau nom de l'organisation (1-255 caractères). Le slug est automatiquement régénéré. |
| `settings` | object | Non | Paramètres de l'organisation (JSON libre). |

### Exemple de requête

```bash
curl -X PUT https://api.dispatchkit.io/api/v1/organizations/b2c3d4e5-f6a7-8901-bcde-f12345678901 \
  -H "Authorization: Bearer {token}" \
  -H "Content-Type: application/json" \
  -d '{
    "name": "Transport Dupont SAS"
  }'
```

### Réponse `200 OK`

```json
{
  "id": "b2c3d4e5-f6a7-8901-bcde-f12345678901",
  "name": "Transport Dupont SAS",
  "slug": "transport-dupont-sas",
  "created_at": "2024-01-15T09:00:00Z",
  "updated_at": "2024-01-15T10:00:00Z"
}
```

### Erreurs

| Code HTTP | Description |
|-----------|-------------|
| `400` | Données invalides (ID invalide, nom trop long) |
| `403` | Permissions insuffisantes (non-admin) |
| `404` | Organisation introuvable |
