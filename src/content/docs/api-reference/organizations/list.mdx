## GET /organizations

Retourne la liste de toutes les organisations auxquelles appartient l'utilisateur authentifié, avec son rôle dans chacune.

### Exemple de requête

```bash
curl https://api.dispatchkit.io/api/v1/organizations \
  -H "Authorization: Bearer {token}"
```

### Réponse

```json
{
  "organizations": [
    {
      "id": "org_01J5XKTY4H3GNQV2",
      "name": "Transport Dupont SAS",
      "slug": "transport-dupont-sas",
      "role": "owner",
      "current": true,
      "members_count": 12,
      "created_at": "2024-01-01T00:00:00Z"
    },
    {
      "id": "org_01J5XKTY4H3GNQV3",
      "name": "Transport Dupont Lyon",
      "slug": "transport-dupont-lyon",
      "role": "admin",
      "current": false,
      "members_count": 8,
      "created_at": "2024-01-10T00:00:00Z"
    }
  ]
}
```

<Aside type="note">
  Le champ `current: true` indique l'organisation active dans le contexte du token JWT fourni. Pour changer d'organisation active, utilisez l'endpoint `/organizations/{id}/switch`.
</Aside>
