## GET /auth/me

Retourne le profil de l'utilisateur associé au token JWT fourni. Utile pour vérifier la validité d'un token et récupérer le contexte utilisateur au démarrage de l'application.

### Headers

| Header | Type | Requis | Description |
|--------|------|--------|-------------|
| `Authorization` | string | Oui | `Bearer {token}` |

### Exemple de requête

```bash
curl https://api.dispatchkit.io/api/v1/auth/me \
  -H "Authorization: Bearer {token}"
```

### Réponse

| Champ | Type | Description |
|-------|------|-------------|
| `id` | string | Identifiant unique de l'utilisateur. |
| `email` | string | Adresse email de l'utilisateur. |
| `first_name` | string | Prénom. |
| `last_name` | string | Nom de famille. |
| `organizations` | array | Liste des organisations auxquelles l'utilisateur appartient. |
| `organizations[].id` | string | Identifiant de l'organisation. |
| `organizations[].name` | string | Nom de l'organisation. |
| `organizations[].role` | string | Rôle de l'utilisateur : `owner`, `admin`, `manager`, `operator`. |
| `organizations[].current` | boolean | Indique si c'est l'organisation active du token courant. |
| `created_at` | string | Date de création du compte au format ISO 8601. |
