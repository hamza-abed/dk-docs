## POST /auth/register

Crée un nouveau compte utilisateur. L'utilisateur devient automatiquement `owner` d'une nouvelle organisation créée lors de l'inscription.

### Body

| Paramètre | Type | Requis | Description |
|-----------|------|--------|-------------|
| `email` | string | Oui | Adresse email de l'utilisateur. Doit être unique sur la plateforme. |
| `password` | string | Oui | Mot de passe. Minimum 8 caractères, doit contenir au moins une majuscule et un chiffre. |
| `first_name` | string | Oui | Prénom de l'utilisateur. |
| `last_name` | string | Oui | Nom de famille de l'utilisateur. |
| `organization_name` | string | Oui | Nom de la nouvelle organisation à créer (ex. : "Transport Dupont SAS"). |

### Exemple de requête

```bash
curl -X POST https://api.dispatchkit.io/api/v1/auth/register \
  -H "Content-Type: application/json" \
  -d '{
    "email": "marie@transport-dupont.fr",
    "password": "MotDePasse2024!",
    "first_name": "Marie",
    "last_name": "Dupont",
    "organization_name": "Transport Dupont SAS"
  }'
```

### Réponse

| Champ | Type | Description |
|-------|------|-------------|
| `token` | string | Token JWT valable 24 heures pour authentifier les prochaines requêtes. |
| `refresh_token` | string | Token de renouvellement valable 30 jours. |
| `user` | object | Informations de l'utilisateur créé. |
| `user.id` | string | Identifiant unique de l'utilisateur. |
| `user.email` | string | Adresse email. |
| `user.first_name` | string | Prénom. |
| `user.last_name` | string | Nom de famille. |
| `user.organization_id` | string | ID de l'organisation créée. |
| `user.role` | string | Rôle dans l'organisation : `owner`. |
