## POST /auth/logout

Révoque tous les refresh tokens actifs de l'utilisateur. L'access token reste valide jusqu'à son expiration naturelle — côté client, il doit être supprimé immédiatement.

### Headers

| Header | Requis | Description |
|--------|--------|-------------|
| `Authorization` | Oui | `Bearer <access_token>` |

### Body

Aucun corps requis.

### Exemple de requête

```bash
curl -X POST https://api.dispatchkit.io/api/v1/auth/logout \
  -H "Authorization: Bearer {token}"
```

### Réponse `200 OK`

```json
{
  "message": "Logged out successfully"
}
```

### Erreurs

| Code HTTP | Description |
|-----------|-------------|
| `401` | Token manquant ou invalide |
| `500` | Erreur interne lors de la révocation des tokens |
