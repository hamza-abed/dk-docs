## POST /auth/register

Crée un nouveau compte utilisateur. L'utilisateur devient automatiquement `owner` d'une nouvelle organisation créée lors de l'inscription.

### Body

<ParamField body="email" type="string" required>
  Adresse email de l'utilisateur. Doit être unique sur la plateforme.
</ParamField>

<ParamField body="password" type="string" required>
  Mot de passe. Minimum 8 caractères, doit contenir au moins une majuscule et un chiffre.
</ParamField>

<ParamField body="first_name" type="string" required>
  Prénom de l'utilisateur.
</ParamField>

<ParamField body="last_name" type="string" required>
  Nom de famille de l'utilisateur.
</ParamField>

<ParamField body="organization_name" type="string" required>
  Nom de la nouvelle organisation à créer (ex. : "Transport Dupont SAS").
</ParamField>

### Exemple de requête

```bash
curl -X POST https://api.dispatchkit.io/api/v1/auth/register \
  -H "Content-Type: application/json" \
  -d '{
    "email": "marie@transport-dupont.fr",
    "password": "MotDePasse2024!",
    "first_name": "Marie",
    "last_name": "Dupont",
    "organization_name": "Transport Dupont SAS"
  }'
```

### Réponse

<ResponseField name="token" type="string">
  Token JWT valable 24 heures pour authentifier les prochaines requêtes.
</ResponseField>

<ResponseField name="refresh_token" type="string">
  Token de renouvellement valable 30 jours.
</ResponseField>

<ResponseField name="user" type="object">
  <Expandable title="Champs">
    <ResponseField name="id" type="string">Identifiant unique de l'utilisateur.</ResponseField>
    <ResponseField name="email" type="string">Adresse email.</ResponseField>
    <ResponseField name="first_name" type="string">Prénom.</ResponseField>
    <ResponseField name="last_name" type="string">Nom de famille.</ResponseField>
    <ResponseField name="organization_id" type="string">ID de l'organisation créée.</ResponseField>
    <ResponseField name="role" type="string">Rôle dans l'organisation : `owner`.</ResponseField>
  </Expandable>
</ResponseField>
